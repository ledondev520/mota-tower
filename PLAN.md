# PLAN — 魔塔 · Mota（15 层经典数值魔塔）

> 若本计划发生重大变更，必须同步更新本文件与 TASKS.md。

## 目标

交付一款**可直接游玩**、逻辑与数值严格符合经典魔塔常理的网页魔塔游戏：

- 双击 `index.html` 即可离线游玩（零依赖、零构建、无外部资源）。
- 同时部署 Vercel，提供线上可玩链接。
- 数值可通关性由 `tools/balance_check.mjs` **机器验证**（全塔模拟到击败魔王），不靠主观感觉。

## 核心规则（经典魔塔约定）

1. 确定性战斗：勇士先手，轮流互击。
   - 勇士每击伤害 `= 勇士攻击 - 怪物防御`（≤0 则无法战胜）。
   - 怪物每击伤害 `= max(0, 怪物攻击 - 勇士防御)`。
   - 战斗总伤害 `= (ceil(怪物HP / 勇士每击) - 1) × 怪物每击`。
2. 特殊能力：`先攻`（怪物多打一轮）、`魔攻`（无视防御）、`连击`（每轮攻击两次），全部纳入伤害预览公式。
3. 资源体系：黄/蓝/红三色钥匙与门；红药水 +200 / 蓝药水 +500 / 圣水 +2000；红宝石 +3 攻 / 蓝宝石 +3 防；三档剑盾装备；金币商店（价格递增防刷）+ 钥匙商店。
4. 致死战斗（预计伤害 ≥ 当前生命）直接拦截，杜绝误操作暴毙。

## 关键取舍

- **13×13 格地图 / 15 层**：比经典 11×11 略宽，房间制骨架（3×3 房间网格 + 门/怪物卡口），空间叙事清晰且便于机器校验连通性。
- **验证器用「贪心通关模拟」而非全状态搜索**：魔塔最优解是 NP 难问题；贪心策略（免费道具→零伤怪→低伤怪→商店兜底）证明"一个稳健玩家可通关"，即满足"数值合理"的工程定义。
- **像素画程序化生成**：调色板 + 字符矩阵定义精灵，同形换色复用（史莱姆/蝙蝠/骷髅等家族），无任何外部图片，file:// 可玩。
- **音效 WebAudio 合成**：无音频文件，首次交互后初始化（符合浏览器自动播放策略）。
- **跨浏览器存档用「存档码」而非云端账号**：纯静态站无后端；v2 差异式存档（只记与初始地图不同的格子）经 deflate+base64url 压成约 300 字符的码，复制/下载即可在任意浏览器恢复，离线可用、零隐私成本。
- **触屏操控双模式**：摇杆（拖拽连走，方向变更即时响应）与方向键（长按连走）一键切换并持久化偏好；统一 Pointer Events，桌面鼠标同样可用。
