## 做了什么

从零交付一款可直接游玩的 15 层经典魔塔（`index.html` 双击即玩 + GitHub Pages 线上版）：

1. **战斗公式模块**（`js/formulas.js`）：经典确定性战斗 + 先攻/魔攻/连击，8 个单元测试。
2. **全塔内容**（`js/data.js`）：23 种怪物、15 层 13×13 手工地图、三色门钥匙经济、
   药水宝石装备、3 家商店（递增定价防刷）、3 位剧情 NPC。
3. **数值验证器**（`tools/balance_check.mjs`）：结构校验 + 资源账本 + 跨层 BFS 贪心通关模拟，
   经 6 轮数值迭代调至全绿（详见 METRICS.md 曲线）。
4. **引擎与表现**：事件驱动引擎（移动/战斗/门/楼梯/楼传/商店/存档）、
   程序化像素美术（无外部资源）、WebAudio 合成音效、怪物手册、地图伤害预览角标、
   三槽存档 + 自动存档、触屏方向键、胜利结算评分。

### 第二轮（同日）：手机端与跨浏览器存档

5. **触屏操控**：虚拟摇杆（拖拽连走、方向变更即时响应）与屏幕方向键（长按连走）
   一键切换并记住偏好；右侧手册/楼传快捷圆钮；统一 Pointer Events，桌面鼠标可用。
6. **移动端布局**：≤900px 或触屏设备启用紧凑 HUD（层/四属性/钥匙/八按钮网格）、
   地图自适应视口、iOS safe-area 适配、禁用双击缩放误触。
7. **跨浏览器存档**：存档升级为 v2 差异式（只记与初始地图不同的格子，兼容旧档）；
   「导出存档」生成 deflate+base64url 压缩存档码（约 300 字符，可复制/下载 .txt），
   任意浏览器标题页「导入存档」粘贴/上传即可原样继续；换层/战斗/开门/购买后自动存档。
8. **标题页三入口**：新的冒险 / 继续冒险（自动读取全部槽位中最新记录并显示摘要）/ 导入存档。

## 关键取舍

- **验证器用贪心模拟而非全状态搜索**：魔塔最优化是指数级问题；贪心"稳健玩家"通关
  即证明塔可通关（充分性），配合钥匙供给宽松化设计（黄 27/22）覆盖普通玩家行为空间。
- **像素画程序化生成**：牺牲一些美术上限，换取零资源依赖与 file:// 直开。
- **致死战斗拦截**：偏离经典"允许送死"，换取更好的现代体验；魔塔中送死从不是最优策略，
  故不影响任何通关路线。
- **贪心模拟与游戏共用同一份 data/formulas**（vm 加载），杜绝"验证的与玩到的不一致"。

- **跨浏览器存档选「存档码」而非云端账号**：纯静态站无后端，账号体系成本与隐私负担高；
  存档码离线可用、可粘贴可存文件，即达成"不同浏览器续玩"的目标。

## 遗留与建议

- 音效为极简合成风格；如需 BGM 可后续加 WebAudio 音序器。
- 存档现为 v2 差异式（读档兼容 v1 全量档）；未来改地图结构需提升版本并写迁移。
- 存档码依赖手动复制/传输；若将来要"自动同步"，可加接 Supabase 等免费后端。
